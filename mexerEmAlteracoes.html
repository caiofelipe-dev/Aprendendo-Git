Revertendo commits

Para reverter um commit no Git, você pode usar o comando git revert <hash> seguido do hash do commit que deseja reverter. Isso criará um novo commit que desfaz as alterações do commit original.

Resetando commits

Para apontar a branch atual pra um commit específico (removendo os posteriores, caso não contenham branch) basta usar git reset <hash> seguido do hash do commit que deseja reverter. Isso criará um novo commit que desfaz as alterações do commit original.

Use --hard pra descartar alterações feitas.

Use --soft pra manter as alterações dentro do índice(index) já adicionado, marcados como staged changes, prontos para serem commitados.

Use --mixed pra manter as alterações dentro do índice(index), mas fora da árvore de trabalho(diretório local), marcados como changes.
